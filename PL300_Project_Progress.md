- [ ] Dataflow -- skipped; know location (My workspace > New > Dataflow) and concept (reusable Power Query in cloud)

#### Key Learnings -- Phase 4
- **RLS filter builder UI:** enter value without quotes (e.g. `Female`); DAX editor requires quotes (`[gender] = "Female"`)
- **"Dataset" = "Semantic model"** in Power BI Service -- same thing, renamed
- **CSV sources require gateway** for scheduled refresh -- cloud sources (Azure SQL, SharePoint Online) do not
- **Sensitivity labels** require Microsoft Information Protection (MIP) licenses -- may be unavailable in some tenants
- **(Blank) under RLS** in bar chart is expected behavior -- role filter on Dim_Patient can leave unmatched DiagnosisGroup values
- **Dashboard vs Report:** dashboards are pinned tile collections; reports are the full authored experience; dashboards do not respect RLS in the same way

## Star Schema Design

```
diabetic_data_staging (raw source -- not loaded to model)
        |
        ├── Dim_Patient (patient_nbr PK -- Text)
        ├── Dim_PrimaryDiagnosis (diag_1 PK + DiagnosisGroup)
        ├── Dim_A1cResult (Index surrogate key + A1cTested)
        ├── Dim_AdmissionType (from IDS_mapping rows 1-8)
        ├── Dim_DischargeDisposition (from IDS_mapping)
        ├── Dim_AdmissionSource (from IDS_mapping)
        └── Fact_Encounters (encounter_id Int64 PK, all FK columns as Text)

_Measures (unrelated container table -- 19 DAX measures)
Dim_Date (unrelated -- no date column in fact table)
```
